Does this résumé actually fit the job?

A small Retrieval-Augmented Generation demo that reads a job description and a stack of résumés, ranks them by real semantic similarity, and explains — in plain language — why each top match earned its spot. Built by a recruiter who also writes the retrieval code.

RAG SENTENCE-TRANSFORMERS GRADIO RUNS FREE, NO SERVER

01 What it does

STEP 1

Embeds

The job description and every résumé are converted to vectors with a sentence-transformer model — meaning is captured, not just keywords.

STEP 2

Retrieves

Résumés are ranked by cosine similarity to the job vector. This is the "R" in RAG — pulling the most relevant candidates before any generation happens.

STEP 3

Explains

Each top match gets a short, grounded explanation — LLM-generated if a token is configured, or an honest keyword-overlap fallback if not. Nothing is invented.

02 Try the lightweight version, right here

This in-page demo runs entirely in your browser using TF-IDF weighted cosine similarity — no model download, no server, so it can live on a free static page. The full app (linked below) swaps this for real transformer embeddings and optional LLM explanations.
